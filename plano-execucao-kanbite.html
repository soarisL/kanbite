Semana 0 — Antes de começar

Configuração do GitHub

1. Repositório

  • Criar repo público: kanbite
  • Adicionar README.md com visão geral
  • Criar branch main (protegida)
  • Criar branch develop (base)
  • Regra: PR obrigatório para main
  • Regra: 1 revisor obrigatório por PR

2. GitHub Projects

  • Criar Project: "KanBite Development"
  • Layout: Board (Kanban)
  • Adicionar 5 colunas (ver seção abaixo)
  • Ativar campo: Assignee, Sprint, Priority
  • Ativar campo: Label, Milestone, Size
  • Criar 4 Milestones = 4 Sprints

3. Labels a criar

  • 🔴 priority: alta — #D85A30
  • 🟡 priority: media — #D4A84B
  • 🟢 priority: baixa — #4AAD85
  • 🔵 type: feature — #5B8FD4
  • 📄 type: doc — #C46B9A
  • 🧪 type: test — #4AAD85
  • 🐛 type: bug — #D85A30
  • 🏗 type: infra — #7A7268

4. Branches por dev

  • Convenção: feature/dev1-auth
  • Convenção: feature/dev2-kanban
  • Convenção: feature/dev3-metrics
  • Convenção: feature/dev4-ui
  • Convenção: docs/dev5-artefatos
  • Merge: sempre via PR → develop

5. Estrutura de pastas

  • kanbite/app.py — entrypoint
  • kanbite/pages/ — telas Streamlit
  • kanbite/services/ — lógica de negócio
  • kanbite/models/ — entidades Pydantic
  • kanbite/database/ — ORM + repos
  • kanbite/tests/ — pytest
  • docs/ — artefatos PDF

6. Cerimônias do sprint

  • Segunda-feira: Sprint Planning (30min)
  • Diariamente: Daily standup — async no Slack/Discord
  • Formato: O que fiz? O que farei? Bloqueio?
  • Sexta-feira: Sprint Review (30min)
  • Mover cartões no Projects após cada entrega
  • Fechar Issues ao mergar PR
